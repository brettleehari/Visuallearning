01 — The Core Problem

Every API Call Ships the Entire Conversation

Because the LLM remembers nothing, every single API call must include ALL previous context. As conversations grow, this creates a fundamental tension: the context window is finite, but conversations are not.

"The AI model (Dr. Neural) is stateless and remembers nothing. The application layer (Alice) maintains conversation history and feeds it back as context, creating the illusion of memory and continuity."

03 — The Five Alice Strategies

How Apps Build Stateful Magic on Stateless APIs

Just as HTTP needed cookies, sessions, and databases to create the modern web — LLMs need these strategies to create the illusion of intelligent, continuous conversation.

📜

Strategy 1

Full History Replay

Every message ever exchanged is re-sent with each API call. Simple but expensive — costs grow with every turn. Works until you hit the window limit.

📖 Book analogy: "Alice reads Dr. Neural the entire notebook before every visit."
🗜️

Strategy 2

Sliding Window + Summarization

Keep recent messages verbatim, compress older ones into summaries. Like a news editor deciding what makes the front page vs. gets archived.

📖 Book analogy: "Alice summarizes last week's notes but reads today's in full."
🔍

Strategy 3

RAG — Retrieve on Demand

Don't pack everything in. Search for relevant documents only when needed and inject them. Like a smart librarian pulling the right books for your question.

📖 Book analogy: "RAG is just: file processing + database operations + API calls. You know all three."
🧠

Strategy 4

Persistent Memory Layer

Extract key facts (name, preferences, projects) into a separate database. Inject relevant memories per-conversation. The family doctor who remembers you.

📖 Book analogy: "Advanced RAG maintains conversation context like a good family doctor."
🔀

Strategy 5

Server-Side Compaction

The server automatically summarizes old parts of the conversation when approaching limits. The conversation continues indefinitely — early details may become lossy.

📖 Book analogy: "The future of AI is getting better at managing state around stateless intelligence."

06 — The Hidden Danger

Context Rot: When More Is Less

⚠ Performance Degrades as the Window Fills

Research shows that as context windows fill up, models lose accuracy. Below 50% full, they lose tokens in the middle. Above 50%, they lose the earliest tokens. This is why smart context management isn't optional — it's essential.

← Strong recall "Lost in the Middle" zone → ← Strong recall

This is exactly why your book's analogy of the News Editor is so apt — limited space, unlimited information. The editor must choose wisely. Statelessness isn't a limitation — it's a feature, but only if the application layer manages context intelligently.
